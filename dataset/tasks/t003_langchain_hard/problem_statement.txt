Bug: RunnableParallel silently swallows exceptions from sub-chains

When one branch of a RunnableParallel raises an exception, the parallel
executor catches it but merges partial outputs from successful branches
instead of re-raising. The caller never sees the exception.

Minimal repro:
    from langchain_core.runnables import RunnableParallel, RunnableLambda

    def fail(_): raise ValueError("intentional failure")

    chain = RunnableParallel(
        good=RunnableLambda(lambda x: x.upper()),
        bad=RunnableLambda(fail),
    )

    result = chain.invoke("hello")
    # Expected: raises ValueError("intentional failure")
    # Actual:   returns {"good": "HELLO"}  ← silently dropped bad branch

Additionally, the on_chain_error callback is never fired for the failed
sub-chain because the exception is caught in gather_with_concurrency()
before the callback manager is notified.

This affects:
  - sync invoke()
  - async ainvoke()
  - stream() (partial chunks from failed branch are silently dropped)

Files to investigate:
- libs/core/langchain_core/runnables/base.py
    → RunnableParallel.invoke(), ainvoke(), _transform()
- libs/core/langchain_core/runnables/utils.py
    → gather_with_concurrency() — exception aggregation logic
- libs/core/langchain_core/callbacks/manager.py
    → on_chain_error dispatch path
- libs/core/langchain_core/runnables/config.py
    → config propagation to sub-runnables (affects error context)
- libs/core/langchain_core/outputs/llm_result.py
    → output merger (where partial outputs are combined)
